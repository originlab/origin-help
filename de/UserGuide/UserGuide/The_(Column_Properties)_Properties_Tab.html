Kurzname, Langname, Einheiten und Kommentare

Kurzname

Legt den Kurznamen der Spalte fest. Weitere Informationen lesen Sie bitte unter Benennungsregeln für Arbeitsblatt, Spalten und Zellbereich. Dies ist nur bearbeitbar, wenn Zellennotation für Tabellenkalkulationsblatt auf der Registerkarte Eigenschaften des Dialogs Fenstereigenschaften deaktiviert ist.
Langname

Legt den Langnamen der Spalte fest. Weitere Informationen lesen Sie bitte unter Benennungsregeln für Arbeitsblatt, Spalten und Zellbereich.
Einheiten

Legt die Einheiten für die Spalte fest. Einheiten sind nur zur Referenz und werden bei keiner Analyse verwendet.
Kommentare

Legt die Kommentare für die Spalte fest. Kommentare können mehrere Zeilen lang sein.

Breite

Spaltenbreite

Geben Sie die Breite für eine Spalte in dieses Textfeld ein.
Auf alle anwenden

Wenn Sie die im Textfeld Breite festgelegte Breite auf alle Spalten anwenden möchten, aktivieren Sie das Kontrollkästchen Auf alle anwenden.

Um die Breite der Arbeitsblattspalte beim Import zu steuern, siehe die LabTalk-Systemvariable @ISC.

Optionen

Diagrammzuordnung

Die Diagrammzuordnung einer Spalte bestimmt, wie die ausgewählten Daten beim Standardzeichnen und von den Analyseoperationen behandelt werden. Die Auswahlliste Diagrammzuordnung enthält die Optionen X, Y, Z, X Fehler, Y Fehler, Beschriftung, Gruppe oder Subjekt (verwendet bei der ANOVA und einigen statistischen Tests) sowie Ignorieren. Nachdem Sie mit OK bestätigt haben, zeigen die Spaltenüberschriften die neuen Zuordnungen in Klammern nach dem Kurznamen der Spalte an. Ignorierte Daten werden nur in der Spalte Kurzname angezeigt).

Hinweis: Die Bedienelemente der interaktiven Datenauswahl im Dialog Diagrammeinstellungen und in vielen Analysedialogen ermöglichen es, die Diagrammzuordnung zu ignorieren.

Format

Numerisch

Akzeptiert nur numerische Werte.
Text

Akzeptiert jedes alphanumerische Zeichen. Werte in einer Textspalte werden als Text behandelt und können in Berechnungen nicht verwendet werden, außer unter besonderen Umständen, wenn eine Spalte zusätzlich als Kategorisch angegeben ist.

Per Standard kann eine Textzelle 49 Zeichnen enthalten. Dies wird aber bis maximal 799 Zeichen erhöht, wenn (a) mehr Zeichnen in die Zelle eingegeben werden oder (b) der Wert von wks.col.tWidth geändert wird. Im Gegensatz dazu kann eine Text- und Numerische Zelle bis zu 3799 Zeichen enthalten kann.
Zeit

Origin speichert intern die Zeitdaten als Tage, wobei der Bruchteil einer Zahl dem Teil eines Tages entspricht. Die Anzeige dieser Daten in einer Spalte wird von der Einstellungsmöglichkeit Anzeige gesteuert. Die Eingabe von Daten in eine Zeitspalte oder die Umwandlung von Textdaten in Zeit sollte der Formatierung in der Einstellung Anzeige entsprechen.
Datum

Origin speichert seit dem 1. Januar, 4713 BCE intern die Datumsdaten als Tage, wobei der Bruchteil einer Zahl dem Teil eines Tages entspricht. Die Anzeige dieser Zahl in einer Spalte wird von der Einstellungsmöglichkeit Anzeige gesteuert. Die Eingabe von Daten in eine Datumsspalte oder die Umwandlung von Textdaten in Datum sollte der Formatierung in der Einstellung Anzeige entsprechen. Bei der direkten Eingabe von Daten können die Formate von Windows Kurzdatum und Langdatum mit einiger Flexibilität verwendet werden - z.B. 28.01.86, 1-28-1986, 1.28.1986, Jan 28 86, Dienstag Januar 28 1986 werden alle akzeptiert. Origin löst auch Mehrdeutigkeiten im aktuellen Datum auf - wenn Sie z.B. Jan eingeben, wird Origin den aktuellen Tag und das aktuelle Jahr verwenden, um den Eintrag zu vervollständigen.

Beachten Sie, dass Origin seit der Version 2019 zwei alternative Datum-Zeit-Systeme unterstützt -- eines ein wahres, unangepasstes Julianisches Tagessystem und das andere ein "2018"-System, bei dem der "Zeitpunkt Null" = 1. Jan. 2018 00:00 ist.
Month

Akzeptiert nur Monatswerte. Sie können die Monate als Zahlenwerte (1 – 12) oder als Text eintragen. Der Text kann soweit abgekürzt werden, sofern die Buchstabe(n) einen eindeutigen Monat repräsentieren. Zum Beispiel N für November, aber Jun für Juni, um ihn von Jul für Juli unterscheiden zu können.
Wochentag

Akzeptiert nur Wochentage. Geben Sie den Tag als Zahlenwert (0 - 6) oder als Text ein. Der Text kann soweit abgekürzt werden, sofern die Buchstabe(n) einen eindeutigen Tag repräsentieren. Zum Beispiel F für Freitag, aber Di für Dienstag, um ihn von Do für Donnerstag zu unterscheiden.
Text & Numerisch

Dies ist ein Standarddatentyp für alle Spaltenzuweisungen mit Ausnahme der "Beschriftung". Akzeptiert jedes alphanumerische Zeichen. Die Textwerte werden jedoch bei allen Arbeitsblattoperationen (wie Zeichnen der Datenpunkte und Durchführung mathematischer Operationen) als fehlende Werte behandelt, es sei denn, numerische Zellenwerte werden als Text eingesetzt (wie bei der Anzeige der Datenbeschriftung).

Hinweise:

  • Der Standardspaltentyp in Origin-Arbeitsblättern einer jüngeren Version als 6.0 ist Text & Numerisch, im Gegensatz zu älteren Versionen von Origin, in denen Numerisch der Standardspaltentyp mit einem internen Datentyp Double (8) war.
  • Wenn Sie bevorzugen, dass Spalten in einem neuen Arbeitsblatt auf den Typ Numerisch und nicht auf Text & Numerisch gesetzt sind, aktivieren Sie das Kontrollkästchen Standardformat für Spalten = Numerisch im Dialog Optionen (Einstellungen: Optionen: Sonstiges).
Farbe

Setzen Sie das Spaltenformat auf Farbe:

  • Wenn die Nummer 1-24 in die Zellen dieser Spalte gefüllt werden, zeigt die Farbe des Zellenhintergrunds die entsprechende Farbe in der Standardfarbliste von Origin.
  • Wenn der HTML-Farbcode in die Zellen dieser Spalte gefüllt werden, wird die Farbe des Zellenhintergrunds in die entsprechende Farbe geändert.
  • Falls die Zelle ein ungültiger Wert ist, wird sie weiß angezeigt.

Wenn Sie die Hintergrundfarbe der Zellen gezeigt haben, können Sie das Kontrollkästchen Farbwert anzeigen aktivieren, um die Farbwerte als Text in der Spalte zu zeigen. Per Standard ist die Textfarbe auf Auto gesetzt. das bedeutet, dass die komplementäre Farbe der Zellfarbe automatisch gezeigt wird. Sie können auch die Symbolleiste Schriftfarbe verwenden , um eine andere Farbe zu wählen.
Binär

Es sind nur zwei Werte (1 und 0) zugelassen. Standard:

  • Wenn Sie numerische Daten eingeben, wird "0" noch immer als 0 und "1" noch immer als 1 angezeigt. Jegliche Werte, die nicht 0 und 1 sind, werden als fehlend (“--”) angezeigt.
  • Wenn Sie Text eingeben, werden “ja” und “wahr” als 1 angezeigt, “nein” und “falsch” als 0. Jegliche anderen Texte werden als fehlend angezeigt.

Beachten Sie, dass Sie den Wert der Systemvariablen @DBL auf 1 setzen können, um die binären Werte nur auf 1 und 0 zu beschränken (keine fehlend).

Text während der Dateneingabe nicht in numerische Daten konvertieren

Dieses Kontrollkästchen ist nur verfügbar, wenn das Format Text & Numerisch ist. Wenn diese Option aktiviert ist, zeigen die Datensätze des Formats Text & Numerisch numerische Werte als Textelemente. Ansonsten werden numerische Werte mit wissenschaftlicher Notation gezeigt. Wenn Sie zum Beispiel die Zeichenkette "001315472" in einer Spaltenzelle bei deaktiviertem Kontrollkästchen "Text während der Dateneingabe nicht in numerische Daten konvertieren" eingeben, wird die Zeichenkette als 1,31573E6 angezeigt. Wenn dieses Kontrollkästchen jedoch aktiviert ist und Sie die Zeichenkette erneut eingeben, wird die Zeichenkette als 001315472 angezeigt. Beachten Sie, dass das Ändern dieser Einstellung die Anzeige der zuvor eingegebenen Daten nicht beeinträchtigen wird.

Wenn Sie numerische Zeichenketten mit Hilfe von Einzelnes/Mehrere ASCII importieren importieren und sie als Text anzeigen möchten, indem Sie Text während der Dateneingabe nicht in numerische Daten konvertieren aktivieren, sollte Zielspaltenformat beibehalten in der Gruppe Verschiedenes im Dialog impASC aktiviert sein.

Anzeige

Legt fest, wie die Daten in der Spalte angezeigt werden. Die verfügbaren Optionen variieren abhängig von der Auswahl in der Auswahlliste Format.

Verfügbare Optionen für Format = Numerisch oder Text & Numerisch sind:

Dezimal: 1000

(1, 1000, 1E6, 1E9)

Hinweis: Der Schwellenwert zur Konvertierung in die wissenschaftliche Schreibweise wird über die Registerkarte Zahlenformat des Dialogs Optionen festgelegt (Einstellungen: Optionen).
Wissenschaftlich: 1E3

(1E0, 1E3, 1E6, 1E9)

Hinweis: Eine Labtalk-Systemvariable (@ETE) kann verwendet werden, um die am Anfang stehende Null in wissenschaftlicher Notation zu verbergen. Die Standardeinstellung ist 0. Die Werte werden folgendermaßen angezeigt: (1E+00, 1E+03, 1E+06, 1E+09). Durch Festlegen von @ETE=1 werden die anführenden Nullen in der wissenschaftlicher Notation verborgen. Die Werte werden dann folgendermaßen angezeigt: (1E+0, 1E+3, 1E+6, 1E+9). Wählen Sie Einstellungen: Systemvariablen, um den Wert @ETE zu ändern.
Technisch: 1k

(1,0, 1,0k, 1,0M, 1,0G)

Hinweis: Vor Origin 2019 konnten Sie eine Spalte mit dem Format Text & Numerisch zwingen, technische Notation, die direkt eingegeben wurde, als numerisch zu behandeln. Dies taten Sie, indem Sie Wert von @EF = 1 gesetzt haben. In Origin 2019 und höheren Versionen können Sie die technische Notation direkt in eine Spalte mit dem Format Text & Numerisch eingeben und dann das Format in Numerisch ändern. Eine Tabelle der unterstützten metrischen Suffixe wird unten auf dieser Seite aufgelistet.

Eine Anleitung, wie Sie den Wert einer LabTalk-Systemvariablen ändern, finden Sie in dieser FAQ.
Dezimal: 1.000

(1, 1.000, 1E6, 1E9)

Hinweis: Der Schwellenwert zur Konvertierung in die wissenschaftliche Schreibweise wird über die Registerkarte Zahlenformat des Dialogs Optionen festgelegt (Einstellungen: Optionen).
Benutzerdefiniert

Wenn Sie Format = Benutzerdefiniert wählen, haben Sie mehr Optionen, wie Sie Ihre Daten im Arbeitsblatt anzeigen. Siehe weitere Informationen im nächsten Abschnitt Benutzerdefinierte Anzeigeoptionen in Origin.

Hinweis: Jedes benutzerdefinierte Format wird in Ihrem Projekt gespeichert und sollte auf dieselbe Weise angezeigt werden, unabhängig davon, wer angemeldet ist oder welcher Computer verwendet wird.


Wenn Format = Zeit:

Zeit Wählen Sie eine der Standardoptionen für Zeit.

Wenn keine der Standardformate Ihre Anforderungen erfüllt, wählen Sie Benutzerdefinierte Anzeige und erstellen Sie eine benutzerdefinierte Zeichenkette mit
Origins Spezifikationssymbolen für Datums- und Zeitformate.

Die Zeitformate, die mit "DDD" beginnen, sind IRIG-Zeitformate (IRIG = Inter-Range Instrumentation Group). Weitere Informationen finden Sie unter IRIG-Zeit.


Wenn Format = Datum:

Datum Wählen Sie eine der Standardoptionen für Datum.

Wenn keine der Standardformate Ihre Anforderungen erfüllt, wählen Sie Benutzerdefinierte Anzeige und erstellen Sie eine benutzerdefinierte Zeichenkette mit
Origins Spezifikationssymbolen für Datums- und Zeitformate.


Wenn Format = Monat:

Month Wählen Sie eine von drei Optionen für die Monatsanzeige.


Wenn Format = Wochentag:

Wochentag Wählen Sie eine von drei Optionen für die Wochenanzeige.

Alle angemessen formatierten Datums-/Zeitdaten werden im Arbeitsblatt als Julianische Tageswerte des Typs Double(8) gespeichert. Dies ist unabhängig von der Option Anzeige, die Sie wählen können. Daher können Sie frei zwischen Datums-/Zeitformaten wechseln, einschließlich Bruchsekunden, Tag der Woche, Monat und zurück zu Bruchsekunden, ohne dass Daten verloren gehen.



Wenn Format = Farbe:

Farbwert anzeigen Legen Sie fest, ob eine Füllfarbe in der Zelle gezeigt wird. Optional zeigen Sie den Origin-Farbindex oder den HTML-Farbcode an.

 

Weitere numerische benutzerdefinierte Anzeigeformate

Wenn das Arbeitsblattspaltenformat auf Text & Numerisch, Numerisch oder Datum bzw. Zeit gesetzt ist, können Sie ein benutzerdefiniertes Format zur Anzeige der Daten im Arbeitsblatt festlegen. Gründe dafür könnten sein:

  • Daten in einer benutzerfreundlichen Form anzeigen (z. B. als "12%" vs. "0,12")
  • Daten wie Datums-/Zeitdaten in Ihrem eigenen benutzerdefinierten Format anzeigen und Origin die Daten als Datum/Zeit anstatt als einfache Textdaten erkennen lassen


Diese Darstellung zeigt die gleichen numerischen Werte (Spalte A(X)) mit verschiedenen angewendeten numerischen benutzerdefinierten Formaten.

  • Weitere Informationen über die benutzerdefinierte Anzeige von Datums- oder Zeitdaten siehe Datums- und Zeitangaben im Arbeitsblatt.
  • Weitere Informationen zur benutzerdefinierten Anzeige von numerischen oder Text- & numerischen Daten siehe Benutzerdefinierte numerische Formate.

Stellen

Diese Auswahlliste ist nur verfügbar, wenn das Format entweder Text & Numerisch oder Numerisch ist.

Standard Dezimalstellen

Anzeigen aller Ziffern in einer Arbeitsblattzelle, wie es im Kombinationsfeld Anzahl der Dezimalstellen auf der Registerkarte Zahlenformat des Dialogs Optionen vorgegeben ist (Einstellungen: Optionen).
Setze Dezimalstellen =

Anzahl der Stellen, die nach dem Komma angezeigt werden (setzt den Eintrag in Anzahl der Nachkommastellen auf der Registerkarte Zahlenformat des Dialogs Optionen außer Kraft).

Geben Sie die gewünschte Anzahl (n) der Nachkommastellen im zugehörigen Textfeld ein. Dieser Wert = die maximale Anzahl der nach dem Komma angezeigten Stellen. Erfolgt hier kein Eintrag, wird der Standardwert verwendet (Wert = 5 für Double (8) interne Datentypen).
Signifikante Stellen=

Die Anzahl der angezeigten Stellen Geben Sie den gewünschten Wert in das zugehörige Textfeld ein.

Datentyp

Diese Auswahlliste ist nur verfügbar, wenn das Format Numerisch ist. Es legt die Speicherkapazität in Byte fest, die jedem Wert zugeteilt wird.

Double(8)

Teilt jedem Wert acht Bytes an Speicherkapazität zu. Der Wertebereich ist ±1,7E±308 (15 Stellen).
Real(4)

Teilt jedem Wert 4 Bytes an Speicherkapazität zu. Der Wertebereich ist ±3,4E±38 (7 Stellen).
Short(2)

Teilt jedem Wert 2 Bytes an Speicherkapazität zu. Der Wertebereich geht von -32.768 bis 32.767.
Long(4)

Teilt jedem Wert 4 Bytes an Speicherkapazität zu. Der Wertebereich geht von -2.147.483.648 bis 2.147.483.647.
Char(1)

Teilt jedem Wert 1 Byte an Speicherkapazität zu. Der Wertebereich geht von -128 bis 127.
Byte(1)

Teilt jedem Wert 1 Byte an Speicherkapazität zu. Der Wertebereich geht von 0 bis 255.
uShort(2)

Teilt jedem Wert 2 Bytes an Speicherkapazität zu. Der Wertebereich geht von 0 bis 65.535.
uLong(4)

Teilt jedem Wert 4 Bytes an Speicherkapazität zu. Der Wertebereich geht von 0 bis 4.294.967.295.
Complex(16)

Teilt jedem Wert 16 Bytes an Speicherkapazität zu, wobei 8 Bytes für den Realteil und die anderen 8 Bytes für den Imaginärteil reserviert sind. Der Wertebereich ist ±1,7E±308 (15 Stellen).
Hinweise:
  • Datentypen mit weniger als Double(8) können Probleme mit verschiedenen mathematischen Funktionen verursachen. Aus demselben Grund sollten Sie den Datentyp der Spalten überprüfen, bevor Sie Daten in die Arbeitsblattspalte eingeben oder importieren. Wenn der Datentyp nicht den Daten entspricht, riskieren Sie, die Exaktheit in ihren Zahlen zu verlieren.
  • Double(8) ist der einzige Datentyp, der fehlende Werte unterstützt.

Andere Einstellungen

Benutzerdefinierte Anzeige

Das Kombinationsfeld ist nur verfügbar, wenn Format auf Text & Numerisch oder Numerisch und die Anzeige auf Benutzerdefiniert oder Format auf Datum oder Zeit und Anzeige auf Benutzerdefinierte Anzeige gesetzt ist. Sie können das benutzerdefinierte Anzeigeformat in dieses Textfeld eingeben oder in der Auswahlliste auswählen. Diese Einstellung kann mit dem Projekt gespeichert werden. Daher können andere Anwender, wenn sie das Projekt öffnen, die gleiche Anzeige sehen. Weitere Einzelheiten finden Sie auf den Seiten Tabellen der Origin-Formate und Benutzerdefinierte Datums- und Zeitformate.
Auf alle Spalten rechts anwenden

Wenn Sie die Einstellungen der Gruppe Optionen auf alle Spalten rechts neben der aktuellen Spalte im Arbeitsblatt anwenden möchten, aktivieren Sie dieses Kontrollkästchen.
