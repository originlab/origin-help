Eingabedaten

Form der Eingabedaten
  • Zusammengefasst
Führen Sie den Test mit Hilfe der Zusammenfassungsstatistik und nicht den tatsächlichen Stichprobendaten im Arbeitsblatt durch. Sie können sie manuell im Dialog eingeben.
  • Roh
Führen Sie den Test durch, indem der Datensatz von jeder Gruppe der Stichprobe sich in einer separaten Spalte befindet.
  • Indiziert
Führen Sie den Test auf die Eingabedaten durch, die aus einer Stichprobendatenspalte und einer „Gruppierung“sspalte bestehen. Die Gruppierungsspalte sollte Informationen für die zwei Grundgesamtheiten von Interesse enthalten, indexiert (nach Zeilennummer) auf die Spalte der Stichprobendaten.
Eingabedaten

(Zusammengefasst)
  • Stichprobenumfang von Stichprobe1
Die Gesamtanzahl von Stichprobe1
  • Gesamtvorkommen von Stichprobe1
Die Anzahl der Vorkommen eines Ereignisses in Stichprobe1.
  • Stichprobenumfang von Stichprobe2
Die Gesamtanzahl von Stichprobe2
  • Gesamtvorkommen von Stichprobe2
Die Anzahl der Vorkommen eines Ereignisses in Stichprobe2.
Eingabedaten

(Roh)
  • Stichprobe1
Legen Sie eine Spalte als Datensatz für Stichprobe1 fest.
  • Häufigkeit von Stichprobe1
Die Häufigkeitsspalte fasst die Anzahl der Vorkommen des Werts in der Spalte von Stichprobe1 zusammen.
  • Stichprobe2
Legen Sie eine Spalte als Datensatz von Stichprobe2 fest.
  • Häufigkeit von Stichprobe2
Die Häufigkeitsspalte fasst die Anzahl der Vorkommen des Wert in der Spalte von Stichprobe2 zusammen.
Eingabedaten

(Indexiert)
  • Stichprobe
Legen Sie eine Spalte als Stichprobe fest. Dieses Spalte enthält die Stichprobendaten für zwei Gruppen.
  • Gruppenbereich
Legen Sie die Gruppenspalte fest.
  • Häufigkeit
Die Häufigkeitsspalte fasst die Anzahl der Vorkommen des Werts der Stichprobenspalte in der entsprechenden Gruppe zusammen.
Exposition(en)

Legen Sie den Beobachtungszeitraum für die Mengendaten, wie Zeit, Fläche, Volumen, Anzahl der Elemente usw., fest. Der Standardwert ist 1.


Ereignisrate in Poisson-Modellen

Testdifferenz Die zu testende hypothetische Differenz
Nullhypothese Rate1 - Rate2 = Testdifferenz
Alternativhypothese Optionen:
Rate1 - Rate2 <> Testdifferenz
Eine beidseitige Ereignisrate in Poisson-Modellen, die zu beantworten versucht, ob Rate1 - Rate2 = Testdifferenz.
Rate1 - Rate2 > Testdifferenz
Eine einseitige Ereignisrate in Poisson-Modellen nach oben, die zu beantworten versucht, ob Rate1 - Rate2 größer ist als die Testdifferenz.
Rate1 - Rate2 < Testdifferenz
Eine einseitige Ereignisrate in Poisson-Modellen nach unten, die zu beantworten versucht, ob Rate1 - Rate2 kleiner ist als die Testdifferenz.
Signifikanzniveau Das Signifikanzniveau des Tests
Kondenzintervall(e) Berechnen Sie das Konfidenzintervall für die wahre (Grundgesamtheit) Rate.
Konfidenzniveau(s) in % Legen Sie die Konfidenzniveaus fest, für die die Konfidenzintervalle berechnet werden. Diese Option ist nur verfügbar, wenn Konfidenzintervall(e) aktiviert ist.


Testmethode

Origin verwendet zwei Methoden zum Berechnen der p-Werte und Konfidenzintervalle für Ereignisraten in Poisson-Modellen bei zwei Stichproben:

Exakt Es gibt keine Anforderung an den Stichprobenumfang, wenn die exakte Methode verwendet wird. Sie ist ideal für kleine Stichproben und Ereignisse mit geringer Häufigkeit.
Normal-Approximation Die Approximation der Normalverteilung wird immer berechnet. Die Qualität der Approximation neigt dazu, sich mit zunehmendem Stichprobenumfang zu verbessern.
Gepooltes p für Test verwenden Legen Sie die Verwendung einer gepoolten Schätzung von p für den Test fest, wenn die Testdifferenz 0 ist.
